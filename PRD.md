# Study Agent - 产品需求文档 (PRD)

## 1. 产品愿景

打造一款能够脱离传统教育模式的 AI 学习助手，实现**主观化、直接化**的学习体验。

## 2. 产品形态

Web 网站

## 3. 核心功能模块

### 3.1 用户画像系统

**输入维度：**
- 学习阶段（如：初中/高中/大学/职场）
- 学习方向（如：编程/数学/语言/艺术）
- 学习计划（可用时间、学习周期）
- 学习目标（掌握程度、认证需求）
- 学习节奏（快速/稳健/深入）

**输出：** LLM 生成的结构化用户画像，包含学习能力评估与偏好分析

**存储：** 长期记忆数据库

### 3.2 内容检索系统（查询 Agent）

- 全网搜索相关教程与知识资源
- 多源信息聚合与去重
- 资源质量评估与排序
- 输出带置信度的资源片段

**格式过滤器（增强搜索精准度）：**
- 优先抓取结构化较好的内容：Markdown、教育类 Wiki
- 支持 PDF 解析（使用 PDF 提取工具）
- 支持 GitHub Repositories（README、文档目录）
- 支持视频字幕转录（YouTube Transcript API）
- 可配置格式优先级：`Markdown > Wiki > PDF > GitHub > Video`

### 3.3 知识图谱系统（知识网 Agent）

- 从检索结果中提取知识框架
- 建立知识点之间的关联关系
- **逻辑校验层** - 对比标准教学大纲（Syllabus）进行校准
- 形成可视化的知识网络结构

### 3.4 学习规划系统（规划 Agent）

- 基于知识网和用户画像生成学习轨迹
- 制定个性化学习计划
- 动态调整学习进度

### 3.5 教学生成系统（教学 Agent）

**教学流程：**
1. **问题引入** → 创设情境，引发思考
2. **探索引导** → 带领用户自主思考解决方案
3. **知识教授** → 讲解核心概念与解决方法

**内容约束（防幻觉机制）：**
- 教学案例必须来自 RAG 检索的确切原始素材
- LLM 仅负责组织和呈现，不得编造案例或数据
- 每个教学点需标注参考来源
- 使用结构化 Prompt 限制发挥空间

### 3.6 复习与评估系统（复盘 Agent）

**复习触发条件：**
- 时间因素（艾宾浩斯遗忘曲线）
- 前课程掌握程度
- 知识难度
- 与当前课程的关联度

**评估方式：**
- 生成针对性练习题
- 综合多知识点出题

### 3.7 反馈循环系统

- 收集用户对教学方式的评价
- 分析答题情况与掌握程度
- 了解用户改进期望

### 3.8 动态优化系统

- 根据反馈重新绘制用户画像
- 调整学习路径与节奏
- 持续优化教学内容

## 4. 记忆策略（Token 优化）

| 策略 | 说明 |
|------|------|
| 知识不重复生成 | 已生成的知识内容存入缓存，直接复用 |
| 规划不重复生成 | 学习计划版本化管理，仅在有变化时重新生成 |
| 教学采用长 prompt | 仅在生成教学内容时使用完整上下文 |
| **部分失效协议** | 当用户在某章节卡住太久时，仅重新生成该章节相关的后续规划，而非重绘全图 |

**部分失效协议详细说明：**
- 触发条件：用户在某章节停留时间 > 阈值 或 多次练习失败
- 执行逻辑：定位到受影响的知识节点 → 仅重新规划该节点的后续路径
- 缓存失效：采用细粒度缓存键，如 `plan:{userId}:{chapterId}:v{version}`

## 5. 用户流程

```
用户注册 → 填写学习信息 → 生成用户画像 → 生成知识网 → 制定学习计划
                                                           ↓
 ← 更新画像 ← 分析反馈 ← 收集反馈 ← 完成练习 ← 生成练习 ← 生成教学内容 ← 点击章节
```

## 6. 核心改进点总结

| 问题 | 解决方案 | 位置 |
|------|----------|------|
| LLM 编造案例 | 强制 RAG 检索 + 限制发挥空间 | 教学 Agent |
| 搜索碎片化 | 格式过滤器 (Markdown>Wiki>PDF>GitHub>Video) | 查询 Agent |
| 知识网逻辑不通 | 逻辑校验层对比标准教学大纲 | 知识网 Agent |
| Token 浪费 | 部分失效协议 (局部规划更新) | 记忆策略 |

## 7. 技术决策（Peer Review 确认）

### 7.1 逻辑校验层 - Hybrid 混合模式

- **Rule-based**: 检查 DAG 连通性、检测循环依赖
- **LLM-based**: 语义对比，识别知识缺失

### 7.2 标准教学大纲来源

| 领域 | 数据源 |
|------|--------|
| 通用领域 | Wikipedia Category 树、Curriculum.gov API |
| 编程/IT | Roadmap.sh JSON 数据 |
| 学术 | Open Library API、Arxiv 分类索引 |

### 7.3 后端语言选型

**Node.js 20+ (TypeScript)** - 与前端共享类型定义，LangChain.js 生态成熟
