### 逻辑校验层详细说明

**输入：** 初步生成的知识框架

**校验来源：**
- 公开教学大纲数据库（如教育部课程标准、大学公开课大纲）
- 权威教材目录结构
- 行业认证考试大纲

**校验内容：**
1. **前置依赖检查** - 确保知识点的前置关系正确
2. **逻辑连贯性** - 检测是否存在逻辑跳跃或断裂
3. **完整性检查** - 对比标准大纲，补充缺失的核心知识点
4. **难度梯度** - 确保学习曲线合理

**输出：** 经过校验和修正的知识网络

## 11. 技术决策（已确认）

### 11.1 逻辑校验层实现方式 - Hybrid 混合模式

| 层级 | 实现方式 | 职责 |
|------|----------|------|
| **Rule-based** | 图算法 | 检查 DAG 连通性、检测循环依赖、验证节点入度/出度 |
| **LLM-based** | 语义对比 | 输入"标准大纲摘要"+"生成的知识网"，识别语义缺失 |

**工作流程：**
```
初步知识框架 → Rule-based 结构校验 → LLM-based 语义校验 → 最终知识网
```

### 11.2 标准教学大纲来源

| 领域 | 数据源 |
|------|--------|
| 通用领域 | Wikipedia Category 树、Curriculum.gov API |
| 编程/IT | **Roadmap.sh** JSON 数据（权威 IT 学习路径） |
| 学术 | Open Library API、Arxiv 分类索引 |

### 11.3 后端语言选型 - Node.js

**选型：Node.js 20+ (TypeScript)**

**理由：**
- 与前端 Next.js 共享 TypeScript 定义，全栈开发效率极高
- LangChain.js 生态已成熟，满足 MVP 快速迭代需求
- 适合 I/O 密集型的 Agent 编排场景

**例外情况：**
- 如需硬核机器视觉库（如 OpenCV），可单独起 Python 微服务处理
